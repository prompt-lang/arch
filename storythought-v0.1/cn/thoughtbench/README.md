# ThoughtBench

> ThoughtBench 是 Prompt-Lang 生态中用于**评估、验证和标定智能体思维能力的基准测试体系**。
> 它由两个互补的子体系构成：**TBH（假设验证）** 和 **TBS（标尺工具集）**。

---

## 体系结构

```
thoughtbench/
├── README.md           ← 本文件：体系概述与导航
├── tbh/                ← ThoughtBench-Hypothesis：假设驱动的验证体系
│   └── README.md
└── tbs/                ← ThoughtBench-Set：标准化能力标尺工具
    └── README.md
```

## TBH：ThoughtBench-Hypothesis（假设验证）

**面向问题**：在 AGI 研发过程中，研究者或开发者提出关于 Agent 能力的**假设**，需要系统性地验证这些假设是否成立。

**核心思想**：
- 假设由**人类提出**——研究者根据观察、理论或直觉，提出可验证的命题
- 验证由**系统执行**——PML 工作流自动运行测试，收集证据
- 结果**可复现、可积累**——每次验证的结果构成知识库，指导后续研发

**典型假设示例**：
- "Agent 在 3 轮 Prompt-in-Loop 后的答案质量优于单轮回答"
- "引入 SOF 自身本体后，Agent 在资源受限场景下的决策合理性提升 20%"
- "RBE 规则引擎处理 100+ 规则时的延迟不超过 50ms"
- "启用情景记忆后，Agent 在 10 轮对话中的上下文保持率 > 90%"

详见 [`tbh/README.md`](./tbh/README.md)

## TBS：ThoughtBench-Set（标尺工具集）

**面向问题**：需要一个**标准化、可量化**的工具集来衡量不同 Agent 的思维能力，类似于 IQ 测试之于人类智力。

**核心思想**：
- 提供**分层级的标尺**——从基础能力到高级推理
- 每个标尺包含**标准化测试用例、度量指标、基线数据**
- 支持**自动化运行**——通过 `ppcli benchmark` 命令执行
- 结果**可比较**——不同 Agent、不同版本的思维能力可横向对比

**标尺维度**：

| 维度 | 标尺 ID | 测量目标 |
|------|---------|----------|
| 效率 | `IOE` | 内部推理 vs 外部计算的效率比值 |
| 记忆 | `MEM` | 有记忆 vs 无记忆的任务完成质量 |
| 智能层级 | `ICS` | Local/Global/Federated 规划能力分级 |
| 推理深度 | `RD` | 多步逻辑推理的正确率与步数效率 |
| 自感知 | `SA` | SOF 驱动的自适应决策合理性 |
| 规则处理 | `RE` | RBE 规则匹配的吞吐量与正确率 |

详见 [`tbs/README.md`](./tbs/README.md)

## 使用流程

```
1. 研究者提出假设 ──→ TBH 定义验证方案
2. TBH 引用 TBS 标尺 ──→ 选择合适的能力维度
3. PML 工作流执行 ──→ 自动收集数据
4. 结果分析 ──→ 假设确认/推翻/修正
5. 知识积累 ──→ 记录到 PEM 工程记忆
```

## 与 PML 体系的关系

| PML 组件 | ThoughtBench 中的角色 |
|----------|----------------------|
| **IOE 指标** | 效率维度的核心度量 |
| **Memory Backend** | 记忆维度的数据存储 |
| **SOF** | 自感知维度的能力声明 |
| **RBE** | 规则处理维度的执行引擎 |
| **Storyline** | TBH 验证方案的叙事容器 |
| **PEM** | 验证结果的知识积累 |
| **ppcli benchmark** | TBS 标尺的执行入口 |
